    About QDrop

    QDrop is an open-source QA build distribution platform designed for development teams to upload, manage, and distribute internal builds without relying on Google Play or external services.

    What is QDrop?

    QDrop provides a streamlined workflow for distributing internal APK builds to testers. Developers can upload builds through the web interface, attach metadata such as version, environment, and changelog, and instantly make them available to testers through the companion Android application.

    How It Works

    • • Upload builds using the QDrop Web dashboard.
    • • APK files are stored securely on Cloudflare R2.
    • • Build metadata (version, label, changelog, uploader) is saved in Firebase Realtime Database.
    • • The QDrop Android app fetches this metadata and lists available builds for testers.
    • • Testers can download and install builds directly from the mobile app.

    Core Features

    • • Upload APK builds with version, label, and changelog
    • • Real-time upload progress and status
    • • Generate Build IDs and QR codes for quick access
    • • Manage builds with search and bulk actions
    • • Track storage usage and build statistics
    • • Secure cloud storage powered by Cloudflare R2
    • • Metadata synchronization through Firebase

    Platform Components

    QDrop Web - A dashboard for uploading builds, managing versions, viewing storage usage, and organizing QA releases.

    QDrop Android App - A Jetpack Compose application that allows testers to browse, filter, download, and install builds directly on their devices.

    Technology Stack

    • • Frontend: HTML, TailwindCSS, JavaScript
    • • Backend Storage: Cloudflare R2
    • • Database: Firebase RTDB
    • • Mobile App: Android (Jetpack Compose)

    Open Source

    QDrop is open-source and designed to be easily customizable. You can fork the project, modify the workflow, or integrate it into your internal QA pipeline.
